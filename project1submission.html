Submission Instructions for Project #1 Report

Your report and any other documents should be submitted on Quercus here.

Submit your Report

  1. The report for project #1 should follow the guidelines outlined here.

  2. The report should be an html, pdf, or .docx document generated from either an RMarkdown file or Jupyter notebook (NB: currently Jupyter notebooks don’t have a straight forward way to export a notebook to MS Word format).

Submit Data and other Documents used in your Report

  1. The instructor should be able to easily reproduce your report (i.e., run your Jupyter notebook or Knit your RMarkdown document).

  2. In order to reproduce your report the instructor will need the data you used and any accompanying files (e.g., a picture stored as a .jpg file).

  3. Store the data that you used in your report in the cloud (e.g., Github, Dropbox, Google Drive, etc.). Then you should be able to read in the data by specifying the url of the file.

For example, the file sta2453grading.csv is stored in the Github repository https://github.com/ntaback/STA2453. The url for sta2453grading.csv is https://raw.githubusercontent.com/ntaback/STA2453/master/sta2453grading.csv

In both examples below files starting with https:// will automatically be downloaded.

library(tidyverse)

Convert Jupyter Notebook into a Report without Code Cells

To convert a Jupyter Notebook to html, pdf, or LaTeX consider using nbconvert. For example, to convert movies.ipynb to an html document without code cells the command line syntax for nbconvert is:

jupyter nbconvert --TemplateExporter.exclude_input=True movies.ipynb
